Comparison of SSR, CSR and SSG

What are they

  • SSR: Server-Side Rendering The practice to render HTML file at server side, then serve it to browser through http requests
  • CSR: Client-Side Rendering The practice to render HTML file at client side, by manipulating DOM using Javascript file served from server
  • SSG: Static Site Generator A framework to generate static web site at build time, served usually from a CDN

Famous Examples

  • SSR:
    • Next.js (JavaScript)
    • JSP (Java)
  • CSR:
    • Create-React-App (ReactJS)
    • VueJS
  • SSG:
    • Next.js (able to generate static website)
    • Gatsby.js ( focus more on SSG, comparing to Next.js )
    • Nuxt.js ( VueJS counterpart of Next.js, which is using React as UI framework)
